# GitHubデプロイ連携ルール

## 新規記事ファイルの作成時
- ファイル先頭の**1行目**に必ず以下を記述する（これがないと自動デプロイが機能しない）：
  ```
  <!-- wp_post_id: N -->
  ```
  - `N` はWordPress管理画面の記事編集URLに含まれる数字（例：`post=31756`）
- WPで先に記事を作成してIDを確認してからファイルを作ること

## WP→ローカル同期（プル）
- WP側の記事をローカルに取り込むには `wp_pull` コマンドを使う（`~/.zshrc` に定義済み）：
  ```
  wp_pull マリオット/国内ホテル/南紀白浜マリオットホテル.md
  ```
- 全ファイル一括プルは `wp_pull_all`

# 記事構成
## 1. タイトル
- 【プラチナ宿泊記】ホテル名（特徴）

## 2. 導入部
- 自己紹介（例：「こんにちは、マリオットプラチナ会員のアポロです。」）
- 宿泊概要
- 記事のハイライト
- ↑ この直後に「紹介プログラムボックス」を挿入
- **海外ホテルレビュー記事の場合：楽天トラベル予約ボタン（目次の直前）**
  - `<a class="shiny-btn3" href="リンク先のURL" rel="nofollow" target="_blank">楽天トラベルで〇〇ホテルを予約する</a>`
  - リンク先URLは別途指定（楽天トラベル）
  - 新しいタブで開くように`target="_blank"`を必ず追加する
  - `rel="nofollow"`も必ず追加する
- 続けて目次を作成（H2/H3をリンク形式で）

## 3. ホテル基本情報
- 開業時期（2020年以降なら必ず記載）
- 立地・アクセス
- 駐車場情報

## 4. プラチナ会員特典
- 会員特典一覧
- アップグレード実績
- ラウンジ情報（なければその旨を記載）

## 5. 施設レビュー
- 客室
- レストラン（夕食・ランチ等。朝食は下記「朝食セクション」で別H2にまとめる）
- プール・フィットネス
- アメニティ
- **海外ホテルレビュー記事の場合：楽天トラベル予約ボタンを適切な場所に配置**
  - `<a class="shiny-btn3" href="リンク先のURL" rel="nofollow" target="_blank">楽天トラベルで〇〇ホテルを予約する</a>`
  - 新しいタブで開くように`target="_blank"`を必ず追加する
  - `rel="nofollow"`も必ず追加する

## 5-2. 朝食セクション（必須）
- **H2見出しで独立セクションを設ける**（例：`ウェスティンホテル横浜の朝食`）
- ラウンジ朝食とレストラン朝食が**別会場・別特典**のホテルでは、必ず区別して記載する
  - プラチナ特典の無料朝食（多くはクラブラウンジ）
  - ウェルカム特典の「朝食2人分」（多くはレストランのフルブッフェ）
- 朝食セクションに**最低限含める項目**（公式サイトで確認し、漏れなく記載）：
  - 会場名・フロア（レストラン正式名称）
  - 営業時間（平日／土日祝で異なる場合は分けて記載）
  - 料金（大人・子ども）
  - 無料で食べる方法（プラチナ特典／ウェルカム特典のどちらか、または両方）
  - ウェルカム特典3択がある場合は、レストラン朝食を選ぶべき理由も簡潔に
- **H3でビュッフェ内容をカテゴリー別に紹介**（例：和食／洋食／パン・スイーツ／ライブキッチン／ドリンク）
- ラウンジのタイムスケジュール内に朝食を詳述するのではなく、**レストラン朝食は本セクションに集約**する
- 目次・「本記事で解決できるお悩み」にも朝食へのリンクを入れる

## 6. まとめ
- 総評とおすすめポイント
- **必ず「以上、少しでも参考になれば幸いです。」で締める（宿泊記・一覧記事共通）**
- **海外ホテルレビュー記事の場合：楽天トラベル予約ボタン（「以上、少しでも参考になれば幸いです。」の直後）**
  - `<a class="shiny-btn3" href="リンク先のURL" rel="nofollow" target="_blank">楽天トラベルで〇〇ホテルを予約する</a>`
  - 新しいタブで開くように`target="_blank"`を必ず追加する
  - `rel="nofollow"`も必ず追加する
- 直後に再度「紹介プログラムボックス」を挿入

# 海外ホテルレビュー記事の場合の追加ルール
- 楽天トラベル予約ボタンを最低3箇所に配置する
  - 目次の直前（## 2. 導入部の最後）
  - 施設レビュー内の適切な場所（## 5. 施設レビュー）
  - 「以上、少しでも参考になれば幸いです。」の直後（## 6. まとめ）
- ボタンHTML形式：
  - `<a class="shiny-btn3" href="リンク先のURL" rel="nofollow" target="_blank">楽天トラベルで〇〇ホテルを予約する</a>`
- リンク先URLは別途指定（楽天トラベル）
- テキストは「楽天トラベルで〇〇ホテルを予約する」（〇〇はホテル名）
- 新しいタブで開くように`target="_blank"`を必ず追加する
- `rel="nofollow"`も必ず追加する
- 必要に応じて他にも配置可能だが、最低3箇所は必須
- **本文テキスト内のホテル名にも楽天トラベルのリンクを貼る**
  - 本文中にホテル名が登場する際は、そのホテル名に楽天トラベルのリンクを設定する
  - 例：`<a href="リンク先のURL" rel="nofollow" target="_blank">◯◯マリオットホテル</a>`
  - リンク先URLは別途指定（楽天トラベル）
  - 新しいタブで開くように`target="_blank"`を必ず追加する
  - 過度にリンクを貼りすぎないよう、自然な頻度で配置すること

# 一覧まとめ記事のルール（ベビーベッド・ラウンジ・ホテル一覧等）

## 公開フロー
- 新規記事は必ず**下書き（draft）**で作成する（`--post_status=draft`）
- 最終チェックをユーザーが行った後に公開する。**Claude側で勝手に公開しない**
- 予約投稿日時はユーザーに確認してから設定する

## カテゴリー設定
- 新規記事作成時はWP-CLIの`--post_category`や`wp post term set`を使うが、**存在しないカテゴリーを勝手に作らない**
- 設定前に必ず`wp term list category`で対象のterm_idを確認する
- `wp post term set {ID} category {term_id} --by=id`で設定後、**未分類（term_id:1）が残っていないか確認し、残っていたら同コマンドで上書き除去する**
- **`wp term delete`は絶対に慎重に使う**：term_idと数値名のカテゴリーが混在する場合、意図しないカテゴリーを削除するリスクがある。削除前に必ず`wp term list`で対象を目視確認する

## タイトルルール
- タイトルに**半角スペースを使わない**
- キーワードは「系列」「対応」などの語で自然につなぐ（例：「マリオット系列ベビーベッド対応ホテル完全一覧」）

## 記事構成ルール
- 紹介フォーム（contact-form-7）は**目次の直前**に配置する
- 免責事項（red_box）は紹介フォームの直後に配置する
- 事実に基づくセクションのみ記載する（「ラウンジと朝食特典の違い」のような誤解を招く比較は書かない）

## カテゴリー設定
- `wp post term set {ID} category {term_id} --by=id` で設定する
- 設定後、**未分類（term_id: 1）が残っていないか必ず確認する**（残っていたら `wp post term set` で上書き）
- カテゴリー削除コマンドは慎重に：`wp term delete category {term_id}` はterm_idを番号そのままで解釈するため、意図しないカテゴリーを削除しないよう事前に対象を確認する

## Hotelierカード作成ルール
- 楽天サムネイル画像URLのパターン（必ずこの形式を使う）：
  `https://img.travel.rakuten.co.jp/share/HOTEL/{楽天ホテル番号}/{楽天ホテル番号}.jpg`
- 独自URLや大サイズ画像（`/LARGE/`や`/origin/`パス）はアスペクト比が崩れるため使用しない
- 楽天未掲載ホテルはHotelierカード不要。テキスト説明のみでリンクも貼らない
