# GitHubデプロイ連携ルール

## 新規記事ファイルの作成時
- ファイル先頭の**1行目**に必ず以下を記述する（これがないと自動デプロイが機能しない）：
  ```
  <!-- wp_post_id: N -->
  ```
  - `N` はWordPress管理画面の記事編集URLに含まれる数字（例：`post=31756`）
- WPで先に記事を作成してIDを確認してからファイルを作ること

## WP→ローカル同期（プル）
- WP側の記事をローカルに取り込むには `wp_pull` コマンドを使う（`~/.zshrc` に定義済み）：
  ```
  wp_pull マリオット/国内ホテル/南紀白浜マリオットホテル.md
  ```
- 全ファイル一括プルは `wp_pull_all`

# マリオット固有の執筆ルール

宿泊記の構成・ショートコード規約・品質管理・相互リンク等の共通ルールは
リポジトリ直下の `CLAUDE.md`（特に「ヒルトン vs マリオット：宿泊記の主な違い」表）を参照する。
このファイルにはマリオット固有の参照先と、共通ルールだけでは判断が難しい実務ノウハウのみ記載する。

## 参照先
- クレカ訴求セクション（anker6）の定型文：`マリオット/General/【H2ヘッダー】マリオットカードでお得に泊まろう.md`
- 国内ホテル一覧まとめ記事：`マリオット/General/日本国内マリオット系列ホテル一覧【2026年版】.md`（公開URL：`https://ibis-dallas.com/marriott-hotel-japan`）
- フェアフィールドの場合は `マリオット/General/日本国内フェアフィールドバイマリオットホテル一覧【2026年版】.md`（公開URL：`https://ibis-dallas.com/fairfield-hotel-japan`）も併用

## ウェルカム特典（3択）の書き方
プラチナエリートのウェルカム特典（1,000ポイント／朝食2人分／ウェルカムドリンク等、ホテルにより内訳は異なる）は、
**プラチナ特典の無料朝食（多くはクラブラウンジ）とは別枠**であることを必ず区別して説明する。

- ラウンジ朝食とレストラン朝食が別会場・別特典のホテルでは、「プラチナ特典で無料になるもの」と
  「ウェルカム特典を使わないと無料にならないもの」を明確に分けて書く
- レストラン朝食の方が品数・満足度で上回るケースが多いため、「初めての宿泊なら朝食2人分がおすすめ」等、
  実際に比較した上でのおすすめ選択肢を示す（一般論での断定は避け、体験に基づいて書く）
- ウェルカム特典3択がある場合は、目次・朝食セクション双方で「ラウンジ朝食」と「ウェルカム特典の朝食」を
  別項目として案内する

# 一覧まとめ記事のルール（ベビーベッド・ラウンジ・ホテル一覧等）

宿泊記とは別の記事タイプ（テーマ別のホテル横断まとめ記事）に関するルール。

## 公開フロー
- 新規記事は必ず**下書き（draft）**で作成する（`--post_status=draft`）
- 最終チェックをユーザーが行った後に公開する。**Claude側で勝手に公開しない**
- 予約投稿日時はユーザーに確認してから設定する

## カテゴリー設定
- 新規記事作成時はWP-CLIの`--post_category`や`wp post term set`を使うが、**存在しないカテゴリーを勝手に作らない**
- 設定前に必ず`wp term list category`で対象のterm_idを確認する
- `wp post term set {ID} category {term_id} --by=id`で設定後、**未分類（term_id:1）が残っていないか確認し、残っていたら同コマンドで上書き除去する**
- **`wp term delete`は絶対に慎重に使う**：term_idと数値名のカテゴリーが混在する場合、意図しないカテゴリーを削除するリスクがある。削除前に必ず`wp term list`で対象を目視確認する

## タイトルルール
- タイトルに**半角スペースを使わない**
- キーワードは「系列」「対応」などの語で自然につなぐ（例：「マリオット系列ベビーベッド対応ホテル完全一覧」）

## 記事構成ルール
- 紹介フォーム（contact-form-7）は**目次の直前**に配置する
- 免責事項（red_box）は紹介フォームの直後に配置する
- 事実に基づくセクションのみ記載する（「ラウンジと朝食特典の違い」のような誤解を招く比較は書かない）

## Yadokkoカード作成ルール
- ホテルの予約カードは `[yadokko id="XXXXX"]` ショートコードを使う（旧Hotelier7の`[hotelier id="XXXXX"]`は廃止済み。CLAUDE.md参照）
- サムネイル画像は楽天トラベルのホテル番号から以下のパターンで参照する：
  `https://img.travel.rakuten.co.jp/share/HOTEL/{楽天ホテル番号}/{楽天ホテル番号}.jpg`
- 独自URLや大サイズ画像（`/LARGE/`や`/origin/`パス）はアスペクト比が崩れるため使用しない
- 楽天未掲載ホテルはYadokkoカード不要。テキスト説明のみでリンクも貼らない
