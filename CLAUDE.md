## 記事タイプ別の適用ルール

### 1. ホテル宿泊記（メインコンテンツ）

チェーンに応じて専用ルールを読み込む：

- **ヒルトン** → `ヒルトン/General/writing-rules.md`
- **マリオット** → `マリオット/General/writing-rules.md`
- **IHG・ハイアット・その他** → 本ファイルの共通ルール＋同チェーンの過去記事を参照

### 2. クレジットカード記事

- 対象読者（初心者／上級者）を冒頭で明示
- 年会費・入会特典・ステータス特典は**公式サイトで必ずファクトチェック**
- 紹介リンクは `紹介リンク.md` を参照
- 関連クレカ記事へ `[nlink]` で内部リンク

### 3. 旅行・観光・アフィリエイト記事

- 収益導線は `紹介リンク.md` の対象商品に合わせる
- 体験談ベースで書く（推測・一般論のみの記述は避ける）

### 共通（両チェーン）

- **朝食は独立したH2セクションで必ず詳述**（会場・時間・料金・無料条件・ビュッフェ内容）。ラウンジ内の一言紹介にとどめない
- トマレバリンク使用時は `posted with トマレバ` および tomareba.com へのリンクを**完全削除**
- 海外ホテルレビューは楽天トラベル予約ボタン（`shiny-btn3`）を**最低3箇所**配置
- まとめ末尾は「以上、少しでも参考になれば幸いです。」で締める

## 記事構成テンプレート（宿泊記・新フォーマット）

### 冒頭ブロック

```
1. [box class="green_box" title="本記事で解決できるお悩み"] + 悩みリスト（各H2へのアンカーリンク）
2. [voice] ショートコード（アポロのひとこと）
3. 自己紹介 + 宿泊概要 + ハイライト（<mark>で強調）
4. トマレバリンク（あれば）
5. 紹介プログラムボックス（1回目）
6. 目次【本記事の内容】
```

### 本文ブロック

```
H2: 基本情報（anker1）→ H3: アクセス / 駐車場 / 施設
H2: 会員特典（anker2）→ H3: ウェルカム特典等
H2: 客室（anker3）
H2: ラウンジ・食事（anker4）※ラウンジの時間割・カクテルタイム等
H2: 朝食（anker5）※必須。レストラン朝食をH3で詳述（会場・時間・料金・ビュッフェ内容）
H2: まとめ（anker6）
```

**朝食セクションの必須要素（漏れ防止チェックリスト）**

| 項目 | 記載内容 |
|------|----------|
| 会場 | レストラン正式名称・フロア |
| 時間 | 営業時間（平日／土日祝の違いがあれば分ける） |
| 料金 | 大人・子ども（税込表記） |
| 無料条件 | ダイヤモンド／プラチナ特典、ウェルカム特典の適用先 |
| 会場の区別 | ラウンジ朝食とレストラン朝食が別なら明記（マリオットは特に重要） |
| ビュッフェ内容 | H3またはH4でカテゴリー別（和食・洋食・パン・ライブキッチン・ドリンク等） |
| 目次・お悩みボックス | 朝食セクションへのアンカーリンクを入れる |

### ヒルトン記事のみ追加

```
まとめH2の前: 「ヒルトンステイをもっとお得に！アメックス提携カード…」セクション
導入直後: 「本記事の信頼性」ボックス
```

## WordPress HTML・ショートコード規約

### 見出しアンカー

- H2: `id="anker1"`, `anker2` …
- H3: `id="anker11"`, `anker21` …（親番号+連番）
- 目次リンク形式: `1.<a href="#anker1">見出し</a>` / `　1-1.<a href="#anker11">小見出し</a>`

### よく使うショートコード・ブロック

```html
<!-- 内部リンク -->
[nlink url="https://ibis-dallas.com/記事スラッグ"]

<!-- 紹介ボックス（ヒルトン） -->
[box class="yellow_box" title="ヒルトン紹介プログラム"][contact-form-7 id="d99b16b" title="ヒルトンアメックス紹介リンク"][/box]

<!-- 紹介ボックス（マリオット） -->
[box class="yellow_box" title="マリオット紹介プログラムで申請する"][contact-form-7 id="238d4b4" title="マリオットアメックス紹介リンク"][/box]

<!-- 特典まとめボックス -->
[box class="black_box" title="ホテル名のプラチナ特典"]

<!-- スケジュール等の補足ボックス -->
[box class="glay_box" title="ラウンジ タイムスケジュール"]

<!-- アポロの声 -->
[voice icon="https://ibis-dallas.com/wp-content/uploads/2021/07/ワシアイコン-scaled.jpg" name="アポロ" type="l"]メッセージ[/voice]

<!-- 海外ホテル：楽天トラベル予約ボタン -->
<a class="shiny-btn3" href="楽天トラベルURL" rel="nofollow" target="_blank">楽天トラベルで〇〇ホテルを予約する</a>
```

### ボックスの使い分け

| class | 用途 |
|-------|------|
| `green_box` | 記事で解決できるお悩み、おすすめポイント |
| `yellow_box` | 紹介プログラム、信頼性、注意喚起 |
| `black_box` | 会員特典の一覧まとめ |
| `glay_box` | 時間割・スケジュールなどの参照情報 |

### 段落の書き方

```html
<!-- wp:paragraph -->
<p>一文目。</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>二文目。</p>
<!-- /wp:paragraph -->
```

- 1段落＝1文（句点ごと改行）
- 重要フレーズは `<strong><mark>テキスト</mark></strong>` で強調

## 内部リンク・収益設計

### 内部リンク

- `内部リンクURL.md` に存在するURL**のみ**使用
- 関連ホテル・クレカ記事へ `[nlink]` を自然な位置に配置

### 国内ホテル一覧まとめ記事との相互リンク（ヒルトン・マリオット宿泊記で必須）

国内ホテルの宿泊記が完成したら、**宿泊記 ↔ 一覧まとめ記事**の双方向リンクを必ず設定する。

#### 対象のまとめ記事

| チェーン | リポジトリ内ファイル | 公開URL（スラッグ） |
|----------|----------------------|---------------------|
| ヒルトン | `ヒルトン/General/日本国内ヒルトン系列ホテル一覧【2026年版】.md` | `https://ibis-dallas.com/hilton-hotel-japan` |
| マリオット | `マリオット/General/日本国内マリオット系列ホテル一覧【2026年版】.md` | `https://ibis-dallas.com/marriott-hotel-japan` |
| フェアフィールド（マリオット） | `マリオット/General/日本国内フェアフィールドバイマリオットホテル一覧【2026年版】.md` | `https://ibis-dallas.com/fairfield-hotel-japan` |

#### ① 宿泊記 → まとめ記事（宿泊記側）

宿泊記内に、まとめ記事への `[nlink]` を**最低2箇所**設置する。

1. **基本情報セクション**（「〇〇はヒルトン／マリオット系列ホテルのひとつです」の直後）
2. **まとめセクション**（関連ホテルへの言及がある箇所）

```html
[nlink url="https://ibis-dallas.com/marriott-hotel-japan"]
```

#### ② まとめ記事 → 宿泊記（まとめ記事側）

宿泊記の公開スラッグが確定したら、まとめ記事内の該当ホテルセクション（H4見出し＋トマレバリンクの直後）に、宿泊記への `[nlink]` を追加する。

```html
<!-- トマレバリンクの wp:html ブロックの直後 -->
[nlink url="https://ibis-dallas.com/westin-yokohama"]
```

- まとめ記事に当該ホテルのH4セクションがない場合は、適切な地域・ブランド区分に新規追加する
- 宿泊記が未公開の場合は、公開後にまとめ記事を更新する（スラッグ確定後）

#### 海外ホテル宿泊記について

海外ホテルは国内一覧には含まれないため、双方向リンクの対象外。代わりに、該当チェーンの関連記事（ポイントの使い方、特典記事など）へリンクする。

### 収益の基本方針

- 宿泊記の収益導線は**紹介プログラム（AMEX）**が主
- 新規記事提案時は `紹介リンク.md` のクレカ紹介を意識した集客・収益記事を優先
- 記事案は `ブログ記事案.md` に追記（最終更新日を記載）

### アフィリエイト記事の収益導線

- 外部アフィリエイトリンクは `rel="nofollow noopener"` を必ず付ける
- ボタンリンクは `shiny-btn3` クラスを使用（宿泊記と同じ）
- 記事内に最低2箇所（冒頭・まとめ）、可能なら3〜4箇所配置
- アフィリエイトリンクが変わる場合はmdファイルを直接編集してgit push

## 品質管理・ファクトチェック

### 基本方針

- **品質最優先**（時間をかけてよい）
- 公開情報（公式サイト・プレスリリース）を必ず確認
- 不確実な情報・推測は書かない
- 数値・日付・特典内容は特に慎重に確認

### 記事完成後のチェック（最低2回）

1. **1回目**：記事全体を公式情報・Web検索と照合
2. **2回目**：年会費、入会特典、ステータス特典、無料宿泊特典など重要数値を再確認

### 特に確認が必要な項目

- **朝食**（会場名・フロア・営業時間・料金・無料条件・ラウンジ朝食との区別・ビュッフェ内容）
- レストラン名・営業時間
- アクセス・開業時期
- ラウンジの提供時間・子連れルール
- 会員特典の内容（ウェルカム特典の選択肢など）
- リンク・URLの有効性

## 記事完成後の提案ルール

新規記事を作成したら、チャットで以下を**必ず提案**する：

### 1. ブログタイトル案（3〜5案）

- チェーン宿泊記は `【ダイヤモンド宿泊記】` または `【プラチナ宿泊記】` を先頭に
- ホテル名＋読者の検索意図（特典、ラウンジ、子連れ、アクセス等）を含める
- 35〜45文字前後を目安

### 2. スラッグ案（3〜5案）

- 英小文字・ハイフン区切り（例：`westin-yokohama`）
- 既存記事のスラッグパターンと統一（`内部リンクURL.md` を参照）
- ホテル名＋地域が基本。長すぎない（3〜5単語程度）

## 画像について

- 画像は別途WordPressにアップロードして差し込む
- 記事作成時は `<!-- wp:image -->` ブロックまたはプレースホルダーを配置
- alt属性にはホテル名・施設名・画像の内容を入れる
- 楽天トラベルのホテル画像URLはトマレバ経由で流用可

### 撮影写真のアップロードフロー

フォルダパスを教えてもらえば以下を自動実行する：

```
1. フォルダ内の画像一覧を取得（ls）
2. フォルダ名（ホテル名）＋画像内容の確認
   - チャットに画像を貼ってもらうか
   - 「1枚目：ラウンジ、2枚目：朝食」と教えてもらう
3. 代替テキストを生成（例：「ヒルトン大阪 エグゼクティブラウンジ」）
4. SCPでサーバーに転送
5. wp media import --alt="..." でWPに登録
6. 記事のmdファイルに wp:image ブロックを挿入
7. git push でWP自動反映
```

**ファイル名はIMG_1234.jpgのままでOK**（フォルダ名＝ホテル名で文脈を判断する）

---

### 画像アップロード手順（新規画像の標準フロー）

```bash
# 1. ローカル→サーバーにSCP転送
scp -P 10022 -i ~/.ssh/tokitoki777.key ~/Downloads/ファイル名.png \
  tokitoki777@tokitoki777.xsrv.jp:/home/tokitoki777/uploads/

# 2. WPメディアライブラリに登録（代替テキストを必ず設定）
wp media import ~/uploads/ファイル名.png \
  --post_id=XXXXX \
  --title="画像タイトル" \
  --alt="代替テキスト（画像の内容を具体的に説明）" \
  --porcelain
```

**代替テキストの書き方**
- ホテル画像：「ホテル名 + 施設名・フロア・内容」（例：ヒルトン大阪 エグゼクティブラウンジ）
- 公式スクショ：「サービス名 + 画面内容」（例：スイカVPN料金プラン一覧 月額878円から）
- 図解・グラフ：「何を示しているか」（例：スイカVPN速度比較 日本サーバーへの接続速度が最速）

**アイキャッチ画像の設定**
`--post_thumbnail` オプションは効かない場合があるため、メタデータに直接書き込む：
```bash
wp post meta update POST_ID _thumbnail_id MEDIA_ID
```

## push前の自動チェック（pre-pushフック）

`git push` のたびに `scripts/lint_articles.py` が自動実行され、以下を機械的にチェックする：

- 全変更ファイル共通：`[nlink]` のリンク切れ、トマレバ残骸、外部リンクのrel="nofollow"漏れ、wp_post_id有無（警告のみ）
- **新規追加ファイルのみ**（既存記事の編集では対象外）：国内ホテル宿泊記の必須要件（まとめ記事への双方向nlink・朝食H2セクション・Hotelierカード2枚以上）

エラーがあるとpushが中止される。緊急時は `git push --no-verify` で回避可能。

**新しい環境（clone直後など）でのセットアップ**：
```bash
./scripts/install_hooks.sh
```
`.git/hooks/` はgit管理外のため、cloneし直した場合は上記を一度実行する必要がある。

**既知の注意点**：既存の国内ホテル宿泊記60本中45本が、まとめ記事への双方向nlink要件を満たしていない（2026年7月時点のバックログ）。これは意図的に新規記事のみ厳格チェックとすることで、既存記事の軽微な編集がブロックされないようにしている。バックログの一括見直しは別途対応予定。

## 作業フロー（推奨）

```
1. 記事タイプとチェーンを特定
2. 本ファイル ＋ 該当 writing-rules.md を読む
3. 同フォルダの過去記事2〜3本で文体を確認
4. 公式サイトでファクト収集
5. 記事本文を作成（WordPress HTML形式）
6. 内部リンクURL.md でリンク先を確認・設定
7. 【国内ホテル宿泊記】まとめ記事との双方向リンクを設定
   - 宿泊記 → 一覧まとめ記事（hilton-hotel-japan / marriott-hotel-japan 等）
   - 一覧まとめ記事 → 宿泊記（スラッグ確定後）
8. **朝食セクションの漏れチェック**（H2独立セクション・会場・時間・料金・無料条件・ビュッフェ内容）
9. ファクトチェック 1回目
10. ファクトチェック 2回目
11. タイトル案・スラッグ案をチャットで提案
```

## 新規記事の作成フロー

```
1. WP管理画面で新規投稿を「下書き」で作成しpost_idを取得
   または：wp post create --post_title="タイトル" --post_status=draft --post_name=スラッグ --porcelain
2. mdファイルを作成し、先頭2行にpost_idとタイトルを記載
   <!-- wp_post_id: XXXXX -->
   <!-- wp_title: タイトル -->
3. 記事本文を執筆（WordPress HTML形式）
4. git add → git commit → git push でWPに自動反映
5. アイキャッチ画像を設定：wp post meta update POST_ID _thumbnail_id MEDIA_ID
6. カテゴリー設定：wp post update POST_ID --post_category=TERM_ID
```

**スラッグ変更が必要な場合**
mdファイルのスラッグ情報はWPに自動反映されないため、以下で手動更新：
```bash
wp post update POST_ID --post_name=新しいスラッグ
```

## Hotelierカード（予約ウィジェット）

- 国内ホテル宿泊記には必ず設置（最低2枚：冒頭・末尾）
- 使用形式（`<div class="cstmreba">` は不要）：
  ```
  <!-- wp:shortcode -->
  [hotelier id="XXXXX"]
  <!-- /wp:shortcode -->
  ```
- Hotelier IDの調べ方：
  ```bash
  wp post list --post_type=hotelier_hotel --fields=ID,post_title --posts_per_page=500
  ```
- トマレバリンクが残っている場合はHotelierカードに置換する
